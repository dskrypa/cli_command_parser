The core Command classes that are intended as the entry point for a given program.

class command_parser.commands.BaseCommand(args: Args)[source]

Bases: object

Base class for Commands that provides all of the core functionality. It is generally recommended to extend Command instead unless you need to omit/re-define the --help action or any other behavior that strays from the core functionality.

Primary entry point for parsing arguments, resolving sub-commands, and running a command. Calls parse() to parse arguments and resolve sub-commands, then calls run() on the resulting Command instance. Handles exceptions during parsing using the configured ErrorHandler.

To be able to store a reference to the (possibly resolved sub-command) command instance, you should instead use the above mentioned methods separately.

Parameters

  • argv – The arguments to parse (defaults to sys.argv)

  • args – Positional arguments to pass to run()

  • allow_unknown – Whether unknown arguments should be allowed (default: raise an exception when unknown arguments are encountered)

  • kwargs – Keyword arguments to pass to run()

run(*args, **kwargs)[source]

Primary entry point for running a command. Calls main() and handles exceptions using the configured ErrorHandler. Alternate error handlers can be specified during Command class initialization. To skip error handling, call main() directly or define the class with error_handler=None.

Parameters

  • args – Positional arguments to pass to main()

  • kwargs – Keyword arguments to pass to main()

main(*args, **kwargs) → int[source]

If any arguments were specified that are associated with triggering an action method, then that method is called here. Subclasses can override this method if they have only one action, if they need to override the action handling logic here, or if they have code that should run based on whether an action was executed or not.

Initialization code that is common for all actions should be placed in __init__ instead of overriding main.

Parameters

  • args – Positional arguments to pass to the action method

  • kwargs – Keyword arguments to pass to the action method

Returns

The number of actions that were taken
